Gulp-tool-web

Simply build web, development, testing, release the front static files

View project on GitHub

欢迎

gulp-tool-web 是一个总结整合了gulp，browserify，webpack等前端流行构件工具的一个集合工具。目的是更加方便，快捷的构件前端页面。改项目中，目前整合了前端集成了基本的使用工具，less的css预编译，js的文件的connect、压缩，图片的压缩，文件的发布，基于browserify的模块加载等，部分功能正在完善中。

为什么要使用？

  • 前端工具化，组件化趋势的必需品。
  • 实际开发的痛苦，前端做开发往往没有服务器这个概念，只要一个浏览器就ok，而每次修改一个很小的样式，都需要刷新页面这个操作来查看效果。很费apm，可能连续切图几天，受不了。
  • 文章(http://deshui.wang/%E6%8A%80%E6%9C%AF/2016/01/01/why-need-front-end-build/)
  • 知乎(https://www.zhihu.com/question/30597893)

安装

  • npm install 安装服务中所需要的包。
  • gulp server 启动node的web服务，包括转译less， browser-sync 文件更新自动更新浏览器,browserify js 模块加载,用于开发。
  • gulp imagesmin 压缩图片，对应的目录默认是在 /dist/images 也可以在配置中自行设置。
  • gulp html 发布到生产环境中时，合并，压缩 css，和js 文件。 默认生产的文件目录 /dist/; 如果需要打包，可自行添加

优点

  • less，sass等预编译工具能高效快速的的开发css样式。(如果有兴趣可以看看postcss等)
  • 监听文件修改，自动更新页面，（可能在监听js上有时候不是很灵活）
  • 图片压缩，（目前还只是简单的图片压缩，没有做sprites, 百度fis3在这个方面做的还不错）
  • 前端发布之后，交付给后端开发的文件。
